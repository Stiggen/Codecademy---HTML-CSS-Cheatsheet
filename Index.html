Body Tags

Tag Name Description
<!DOCTYPE html> HTML declaration Lets browser know the file typ ebeing used. MUST BE FIRST LINE OF CODE.
<html></html> HTML Tag Contains all HTML code inside of it and lets the browser know.
<head></head> Head Tag Contains metadata about page content not actually visible on page
<title></title> Title Tag Name of page in browser tab. Contained within Head tags
<link> Link Tag Specifies linking CSS file with required 3 attribute syntax: href="filepath" type="text/css" rel="stylesheet". Self closing tag that must be included in head tag
<style></style> Style Tag Can be included in head tag to specify which elements recieve the styling using actual CSS format inside HTML document
<main></main> Main Tag Encapsulates most of the content on a page. Usually contains header and nav tags
<header></header> Header Tag Contains navigational links or introductory content (h1 - h6)
<nav></nav> Navigation Tag Used to define a block of navigation links (for menus, T.O.C. etc)
<footer></footer> Footer Tag For contact information, copyright information, terms of use, site map, reference to top page links
<section></section> Section Tag Defines elements in a document like chapters, headings or other area of the document with a similar theme
<article></article> Article Tag Content that makes sense on its own such as articles, blogs, magazines, comments etc
<aside></aside> Aside Tag Enhances another element but isn't necessary to understand main theme(bibliographies, endnotes, comments,etc)

Attribute Description

controls Includes things like pause,start, stop for video and audio

Form Tags

Tag Name Description
<form></form> Form Tag Indicates to take input from a user (can contain child elements: h1, p, etc)
<input> Input Tag Nested tag inside Form. Specifies kind of input with type="" attribute and association with lable tag via 'id=""' attribute
<label></label> Label Tag Properly identify a form (contains the 'for=""'
<textarea></textarea> Text Area Tag Includes rows/cols to create large user text input area. Text between tags shows as prefilled in the box

Related Form Attributes (included in opening tag)

Attribute Description
action="" used in Form tag. Determines where information is sent
method="" used in Form tag. Performs assigned HTTP verb on the action
type="" Determines how the the form is rendered on the webpage and what data it accepts (text, passord, number, step, range, checkbox, radio, dropdown, datalist, submit)
"text" Creates field for text
name="" Lables input for when it's sent for processing after "Submit". *Each input can have the same value for name but different , unique IDs to pair with a label. Useful for grouping inputs together.
value="" What is entered in the field from user. If filled in will be pre-filled in the form.
"password" Hides user input and converts to something like an asterisk or dot
"number" Accepts numbers, -, + and .
step="" Used with number. Adds arrows to increase or decrease the number by the step value
"range" Creates a slider w/ min="" and max="" attributes. step="n" assigned value of each change, smaller value means more fluid change
"checkbox" Creates checkboxes. Value assignment helps return information but can't always be sen in the form. That's why we use labels.*
"radio" Like checkboxes, group together by assigning same name but only allow one input.
"dropdown" Creates dropdown menu. Syntax:<select id="category" name="category"> <option value="text"> text </option> </select>
"datalist" Uses type="text" to take user input and filter results in a dropdown fashion or accept new user input. Uses list="" attrigute in input element.
"submit" Creates submit button. Can set value="text to display on button(Submit by default)." When submitted, name attribute fields that accept values and their input are submitted as "name=value" pair.
required Validates the form, In input tag element means form can't be submitted without. Uses min/max to restrict numbers, minlength/maxlength to restrict text, pattern to specify pattern (numbers, characters etc).

CSS Selectors

Selector Description
HTML Attribute id="" / CSS Selector #idname { styling} Meant to style one element only. Overrides class and tag. Most Specific. Use if class selector isn't possible.
.class {styling} References coresponding HTML attribute specifcially. Reusable, meant to be used over many elements. Use if tag selector isn't possible. Can use multiple classes by separating the names in the 'class' attribute in HTML tag with a space. Syntax class ="classname1 classname2"
element tag (ie <p>, <h1>, .classname etc) Least specific. Can refrence multiple in CSS by separating with a comma in declaration. Syntax: h1, h2, p, {styling}
Chaining Selectors: tag.classname {styling} Effects only that specific tag type.
Nested Elements: .classname tag Applies class to specified elements.
